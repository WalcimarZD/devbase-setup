# Contributing

Thank you for contributing to Devbase-setup! We welcome improvements, bug
fixes, and suggestions. Please follow these guidelines to make collaboration
easier.

1. Reporting issues
   - Search existing issues before opening a new one.
   - When reporting a bug, include steps to reproduce, environment, and logs.

2. Feature requests
   - Provide a short description, motivation, and proposed design or example.

3. Development workflow
   - Fork the repository and create feature branches from `main`.
   - Use descriptive branch names: `feat/<short-desc>`, `fix/<short-desc>`.
   - Follow Conventional Commits for commit messages (e.g., `feat:`, `fix:`,
     `chore:`).

4. Pull requests
   - Keep PRs focused and small when possible.
   - Link related issues and provide a clear description of changes.

5. Tests and linting
   - Add tests for new features or bug fixes when applicable.
   - Run linters and formatters before opening a PR.

6. Code of Conduct
   - By participating you agree to follow the `CODE_OF_CONDUCT.md`.

If you need to report a security vulnerability, please use the repository's
security contact rather than public issues.
